## Часть 1. План видов характеристик «ВидыСубконто»

### 1.1 Создание объекта

1. Дерево конфигурации → раздел **Планы видов характеристик**
2. Правый клик → **Добавить**
3. Откроется форма. Вкладка **Основные** активна по умолчанию
4. Заполнить поля:

| Поле | Значение |
|---|---|
| Имя | ВидыСубконто |
| Синоним | Виды субконто |
| Комментарий | Типы аналитики для плана счетов |

### 1.2 Настройка типа значения характеристик

На той же вкладке **Основные** (не переходить на другие вкладки):

- Поле **«Тип значения характеристик»** — по умолчанию стоит «Строка»
- Нажать кнопку **«...»** справа от выпадающего списка
- Откроется редактор типа данных
- В редакторе снять флажок **Строка**
- Установить флажок **СправочникСсылка** и выбрать все 6 типов:

| Тип |
|---|
| СправочникСсылка.Сотрудники |
| СправочникСсылка.Номенклатура |
| СправочникСсылка.Подразделения |
| СправочникСсылка.СтатьиЗатрат |
| СправочникСсылка.Контрагенты |
| СправочникСсылка.Котельные |

- Нажать **ОК**
- Нажать **Далее >** (или **Закрыть** и сохранить через Ctrl+S)

### 1.3 Предопределённые элементы

1. Открыть план видов характеристик **ВидыСубконто** (двойной клик в дереве)
2. Вкладка **Прочее** → кнопка **Предопределённые**
3. Откроется таблица с колонками: **Имя, Код, Наименование, Тип**
4. Уже есть один элемент «Характеристики» — его можно оставить или удалить (он не нужен)
5. Добавить 6 элементов кнопкой **«+»** на панели инструментов:

| Имя | Наименование | Тип |
|---|---|---|
| Сотрудники | Сотрудники | СправочникСсылка.Сотрудники |
| Номенклатура | Номенклатура | СправочникСсылка.Номенклатура |
| Подразделения | Подразделения | СправочникСсылка.Подразделения |
| СтатьиЗатрат | Статьи затрат | СправочникСсылка.СтатьиЗатрат |
| Контрагенты | Контрагенты | СправочникСсылка.Контрагенты |
| Котельные | Котельные | СправочникСсылка.Котельные |

> При добавлении каждого элемента: нажать «+», заполнить Имя и Наименование, в колонке Тип нажать «...» и выбрать нужный справочник.

6. Закрыть окно предопределённых — изменения сохраняются автоматически
7. **Ctrl+S** — сохранить план видов характеристик

### 1.4 Проверка

- [ ] В дереве конфигурации: **Планы видов характеристик** → **ВидыСубконто**
- [ ] Тип значения характеристик — составной (6 ссылочных типов)
- [ ] 6 предопределённых элементов в таблице

## Часть 2. План счетов «Основной»

### 2.1 Создание объекта

1. Дерево конфигурации → **Планы счетов**
2. Правый клик → **Добавить**
3. Вкладка **Основные**:

| Поле | Значение |
|---|---|
| Имя | Основной |
| Синоним | Основной план счетов |
| Комментарий | Бухгалтерский учёт котельного предприятия |

4. Нажать **Далее >** или перейти на вкладку **Субконто**

### 2.2 Настройка субконто плана (вкладка «Субконто»)

1. Перейти на вкладку **Субконто**
2. Поле **«Максимальное количество субконто»** — установить **3**
3. Поле **«Виды субконто»** — нажать кнопку **«...»**
4. В открывшемся окне выбрать план видов характеристик **ВидыСубконто** → ОК
5. В таблице **«Признаки учёта субконто»** появится строка — оставить как есть (настройки суммового/количественного учёта задаются на уровне каждого счёта)

### 2.3 Предопределённые счета

1. Перейти на вкладку **Прочее**
2. Нажать кнопку **Предопределённые**
3. Откроется таблица с колонками: **Имя, Код, Наименование, Вид, Забалансовый, Порядок**
4. Уже есть элемент «Счета» — это корневая группа, счета добавляются внутрь неё

### 2.4 Как добавить счёт

Для каждого счёта:
1. Нажать **«+»** на панели инструментов
2. Откроется форма **«Предопределённый счёт»** с полями:
   - **Родитель** — автоматически «Счета» (или другой счёт, если создаём субсчёт)
   - **Имя** — имя для кода (латиница/кириллица без пробелов)
   - **Код** — номер счёта (10, 20, 26 и т.д.)
   - **Наименование** — текстовое название
   - **Вид** — выпадающий список: Активный / Пассивный / Активный/Пассивный
   - **Забалансовый** — флажок (для наших счетов не ставить)
   - **Порядок** — оставить пустым
   - Таблица **«Признак учёта / Учитывать»** — верхняя (оставить как есть)
   - Таблица **«Вид субконто / Только обороты»** — нижняя, сюда добавляем субконто кнопкой «+»
3. Нажать **ОК**

### 2.5 Как добавить субсчёт

1. В форме предопределённого счёта поле **Родитель** — нажать «...» и выбрать родительский счёт
2. Например, для счёта 10.01: Родитель = счёт 10, Код = 10.01

### 2.6 Как добавить субконто к счёту

В нижней таблице формы счёта:
1. Нажать **«+»**
2. В колонке **«Вид субконто»** нажать «...» → выбрать нужный вид из плана ВидыСубконто
3. Флажок **«Только обороты»** — не ставить (нам нужны остатки)

## Часть 3. Регистр бухгалтерии «БухгалтерскийОсновной»

### 3.1 Создание объекта

1. Дерево конфигурации → **Регистры бухгалтерии**
2. Правый клик → **Добавить**
3. Откроется форма. Вкладка **Основные**:

| Поле | Значение |
|---|---|
| Имя | БухгалтерскийОсновной |Бухгалтерский (основной) |
| Комментарий | Журнал проводок по основному плану счетов |

### 3.2 Привязка к плану счетов и корреспонденция

На вкладке **Основные** (там же):

- Поле **«План счетов»** — нажать **«...»**, выбрать **Основной**
- Флажок **«Корреспонденция»** — **поставить**

> После выбора плана счетов флажок «Корреспонденция» станет активным. Он означает двойную запись (дебет + кредит).

### 3.3 Измерения, Ресурсы, Реквизиты

1. Перейти на вкладку **Данные**
2. Видны три узла в дереве: **Измерения**, **Ресурсы**, **Реквизиты**
3. Выделить нужный узел → нажать **«+»** на панели инструментов → откроется форма нового элемента

**Измерения** — выделить «Измерения», нажать «+»:

| Имя | Синоним | Тип |
|---|---|---|
| Организация | Организация | Строка, длина 100 |

**Ресурсы** — выделить «Ресурсы», нажать «+»:

| Имя | Синоним | Тип | Длина | Точность |
|---|---|---|---|---|
| Сумма | Сумма | Число | 15 | 2 |
| Количество | Количество | Число | 15 | 3 |

**Реквизиты** — выделить «Реквизиты», нажать «+»:

| Имя | Синоним | Тип | Длина |
|---|---|---|---|
| Содержание | Содержание проводки | Строка | 250 |

4. **Ctrl+S** — сохранить

- [ ] Регистр «БухгалтерскийОсновной» в дереве конфигурации
- [ ] Вкладка Основные: план счетов = Основной, флажок Корреспонденция установлен

3. В дереве раскрыть **Реквизиты**, найти **СчётУчёта**
4. Двойной клик по реквизиту → откроется форма редактирования
5. Поле **Тип** → нажать **«...»**
6. В редакторе типа:
   - Снять флажок **Строка**
   - Установить флажок **ПланСчетовСсылка.Основной**
7. ОК → **Ctrl+S**

### Документ-заглушка для регистраторов

Регистры бухгалтерии и накопления требуют хотя бы один документ-регистратор — иначе F7 завершается ошибкой. Создать временный документ:

1. Дерево конфигурации → **Документы** → правый клик → **Добавить**
2. Имя: `ДвижениеПоРегистрам`, Синоним: `Движение по регистрам`
3. Вкладка **Движения** → поставить флажки напротив всех трёх регистров:
   - [x] БухгалтерскийОсновной
   - [x] МатериалыНаСкладе
   - [x] НачисленияСотрудникам
4. **Ctrl+S**

### Финальный шаг — обновление конфигурации базы

1. Меню **Конфигурация** → **Обновить конфигурацию базы данных** (или клавиша **F7**)
2. Если система спрашивает про изменение типов данных — нажать **Принять**
3. Убедиться, что обновление завершилось без ошибок
4. Открыть базу: **Отладка** → **Начать отладку** (или Ctrl+F5)
5. В режиме предприятия: **Операции** → **Планы счетов** → **Основной** — все 14 счетов отображаются
