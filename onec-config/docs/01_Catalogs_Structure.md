## Общие принципы

### Структура окна конфигуратора

- **Слева**: дерево объектов конфигурации (Общие, Константы, Справочники, Документы, ...)
- **По центру**: открытые формы и окна свойств
- **Снизу**: окно сообщений (ошибки, предупреждения, результаты)
- **Справа** (если включено `Окно → Свойства`): окно свойств выделенного объекта

### Универсальный порядок создания справочника

1. Развернуть в дереве узел **Справочники** (одинарный клик на треугольнике слева)
2. **Правый клик на «Справочники»** → **Добавить** (или клавиша **Ins**, или кнопка `+` на тулбаре)
3. Откроется окно редактирования объекта — там несколько вкладок:
   - **Основные** — имя, синоним, комментарий
   - **Иерархия** — иерархичен ли справочник
   - **Владельцы** — для подчинённых справочников (нам не нужно)
   - **Данные** — длина кода/наименования, реквизиты, табличные части
   - **Нумерация** — автонумерация кода
   - **Формы** — формы для UI (можно не трогать — 1С создаст автоматически)
   - **Команды**, **Макеты**, **Ввод на основании**, **Права**, **Интерфейсы**, **Прочее**

### Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| **F7** | Сохранить конфигурацию (без обновления БД) |
| **F11** | Запустить отладку (после обновления БД) |
| **Ctrl+F9** | Запустить 1С:Предприятие в режиме отладки |
| **F9** | Запустить 1С:Предприятие без отладки |
| **Ctrl+S** | Сохранить текущий открытый объект |
| **Ins** | Добавить новый объект в выделенный узел дерева |
| **Del** | Удалить выделенный объект |
| **F2** | Переименовать выделенный объект |
| **Shift+F11** | Сохранить + обновить конфигурацию БД (полный цикл) |
| **Alt+Enter** | Открыть свойства выделенного объекта |

### Когда сохранять / обновлять

- **F7** (Конфигурация → Сохранить конфигурацию) — после каждого справочника
- **Обновление конфигурации БД** (`Конфигурация → Обновить конфигурацию базы данных` или **F7** → согласиться на обновление):
  - После всех справочников — один раз
  - 1С спросит «Принять изменения?» — ответить **Да**, «Реорганизация информации» — **Принять**

### Создание реквизита справочника — пошагово

1. Открыть справочник (двойной клик в дереве)
2. Перейти на вкладку **Данные**
3. В разделе **Реквизиты** нажать кнопку **+ Добавить** (или ПКМ → Добавить)
4. Откроется окно свойств реквизита справа (или внизу) — заполнить:
   - **Имя** — англ. буквами, без пробелов (например, `Podrazdelenie`)
   - **Синоним** — заполняется автоматически из имени, но можно изменить вручную на красивый русский (например, «Подразделение»)
   - **Тип** — кликнуть на `...` справа от поля → откроется окно выбора типа:
     - **Строка** — указать длину
     - **Число** — указать длину и точность
     - **Дата** — указать состав даты (Дата, Время, ДатаВремя)
     - **Булево** — нет настроек
     - **СправочникСсылка.ИмяСправочника** — для ссылок на другие справочники
     - **ПеречислениеСсылка.ИмяПеречисления** — для ссылок на перечисления
   - **Длина** — для строки (по умолчанию 25, увеличить при необходимости)
   - **Точность** — для числа (количество знаков после запятой)
   - **Заполнение** — оставить «Не проверять» по умолчанию
   - **Проверка заполнения** — если реквизит обязательный, поставить «Выдавать ошибку»

### Имя vs Синоним

- **Имя** — внутренний идентификатор, используется в коде. Только латиница, цифры, подчёркивание. Не должно начинаться с цифры.
- **Синоним** — отображается пользователю в UI и в дереве конфигурации. Любые символы.

В этом проекте: имена справочников — английские (Subdivisions, Sotrudniki), имена реквизитов — русские транслитом (Podrazdelenie, Dolzhnost). Синонимы — на русском.

## Перечисления

**ВАЖНО**: перечисления нужно создать ДО справочников, потому что справочник «Должности» ссылается на перечисление `Категории`.

### Перечисление: КатегорииСотрудников (KategoriiSotrudnikov)

**Зачем**: для классификации должностей (Рабочий / Специалист / Руководитель).

**Где создаётся**: в дереве конфигурации — узел **Перечисления** (если узла нет в дереве, он появится после первого добавления).

#### Пошагово:

1. ПКМ на узел **Перечисления** → **Добавить**
2. Вкладка **Основные**:
   - **Имя**: `KategoriiSotrudnikov`
   - **Синоним**: `Категории сотрудников`
3. Вкладка **Данные** → **Значения** → кнопка **+ Добавить** (трижды):

| Имя | Синоним |
|-----|---------|
| `Rabochiy` | `Рабочий` |
| `Specialist` | `Специалист` |
| `Rukovoditel` | `Руководитель` |

4. **F7** — сохранить конфигурацию.

### 1. Подразделения (Subdivisions)

**Назначение**: организационная структура компании (управление, цеха, бригады).

**Иерархический**: ДА (иерархия групп и элементов — чтобы бригады лежали внутри производственного отдела).

#### Свойства справочника

| Вкладка | Свойство | Значение |
|---------|----------|----------|
| Основные | Имя | `Subdivisions` |
| Основные | Синоним | `Подразделения` |
| Иерархия | Иерархический справочник | ✅ Установить флажок |
| Иерархия | Вид иерархии | `Иерархия групп и элементов` |
| Иерархия | Ограничение количества уровней иерархии | ❌ Снять флажок (неограниченно) |
| Данные | Длина кода | `9` |
| Данные | Длина наименования | `100` |
| Данные | Основное представление | `В виде наименования` |
| Нумерация | Автонумерация | ✅ Установить |
| Нумерация | Тип кода | `Строка` |
| Нумерация | Контроль уникальности | ✅ Установить |

#### Реквизиты

Реквизиты не требуются — достаточно стандартных полей **Код** и **Наименование**.

#### Тестовые данные (заполнить в режиме Предприятие)

Иерархическая структура:

```
📁 Управление (группа)
   └── Главное управление (элемент внутри группы)
📁 Производственный отдел (группа)
   ├── Бригада №1 (элемент)
   ├── Бригада №2 (элемент)
   └── Бригада №3 (элемент)
Бухгалтерия (элемент верхнего уровня)
Кадровая служба (элемент верхнего уровня)
```

#### Пошаговые действия в конфигураторе

1. ПКМ на **Справочники** → **Добавить** (или **Ins**)
2. На вкладке **Основные**:
   - Имя: `Subdivisions` (Tab автоматически заполнит Синоним)
   - Изменить Синоним на: `Подразделения`
3. Перейти на вкладку **Иерархия**:
   - Установить флажок **Иерархический справочник**
   - **Вид иерархии**: выбрать `Иерархия групп и элементов`
4. Вкладка **Данные**:
   - Длина наименования: `100`
5. **Ctrl+S** — сохранить объект
6. Закрыть окно справочника (Ctrl+F4)

## Финальные действия после создания всех справочников

### 1. Сохранить конфигурацию

- Меню **Конфигурация → Сохранить конфигурацию** (или **F7**)
- Должно появиться сообщение «Конфигурация успешно сохранена» в окне сообщений (внизу)

### 2. Обновить конфигурацию базы данных

- Меню **Конфигурация → Обновить конфигурацию базы данных** (или **F7** → согласиться на обновление)
- Появится окно **Реорганизация информации** со списком новых таблиц
- Нажать **Принять**
- Дождаться окончания процесса (обычно несколько секунд)

### 3. Запустить 1С:Предприятие для заполнения данных

- Меню **Сервис → 1С:Предприятие** (или **F5**)
- Или **Отладка → Начать отладку** (**F5**)
- Откроется новое окно — 1С:Предприятие в режиме пользователя
- В главном меню сверху должны появиться разделы (Главное, Справочники и т.д.)
- Если нет красивого меню — открывать справочники через **Все функции** (**Сервис → Параметры → Отображать команду «Все функции»** ✅, затем меню **Все функции** → Справочники)

### 4. Заполнить справочники по порядку

**Важен порядок заполнения** — справочники со ссылками заполняются ПОСЛЕ тех, на которые ссылаются:

```
1. KategoriiSotrudnikov (перечисление — уже задано в коде)
2. Subdivisions
3. Dolzhnosti
4. EdinicyIzmereniya
5. VidyNomenklatury
6. Kontragenty
7. StatiZatrat
8. VidyNachisleniy
9. VidyUderzhaniy
10. OrganizaciyaNasha
─── после этих можно заполнять справочники со ссылками ───
11. Sotrudniki (ссылается на Subdivisions, Dolzhnosti)
12. Nomenklatura (ссылается на VidyNomenklatury, EdinicyIzmereniya)
13. Kotelnye (ссылается на Kontragenty)
```

#### Как создать элемент справочника

1. Открыть справочник через меню **Справочники**
2. Кнопка **Создать** на тулбаре формы списка (или **Ins**)
3. Заполнить поля
4. **Записать и закрыть** (или **Ctrl+Enter**)

#### Как создать группу в иерархическом справочнике

1. На тулбаре кнопка **Создать группу** (или **Ctrl+F9**)
2. Заполнить наименование
3. **Записать и закрыть**

## Возможные ошибки

### «Имя содержит недопустимые символы»

- Имя должно начинаться с буквы латинского алфавита или подчёркивания
- Допустимы только латинские буквы, цифры, подчёркивания
- Пример: `Sotrudniki` — ОК, `2Sotrudniki` — нет, `Sotrudniki-1` — нет

### «Не задан тип реквизита»

- Кликнуть на `...` в поле **Тип** — выбрать конкретный тип, нельзя оставлять «Произвольный»

### «Длина строки не задана» / «Длина строки = 0»

- Для типа Строка минимальная длина — 1, по умолчанию 25
- Если оставить 0 — будет ошибка при F7

### «Реквизит ссылается на ещё не созданный справочник»

- Если создаём `Sotrudniki` ДО `Subdivisions` — при выборе типа `СправочникСсылка.Subdivisions` его не будет в списке
- Решение: всегда создавать справочники в указанном порядке
- Если справочник уже создан, но в списке его нет — нажать кнопку **Обновить** в окне выбора типа или закрыть/открыть окно

### Ошибка обновления БД: «Невозможно изменить тип реквизита»

- Происходит, если есть данные и меняется тип на несовместимый
- Решение: очистить данные в реквизите перед изменением типа
- Для нашего случая (День 12): база ещё пустая, ошибки не должно быть

### Конфигуратор не запускается / база не открывается

- Проверить путь к базе: D:\BoilerManagementSystem\onec-config\db
- Запускать конфигуратор от имени администратора
- Если база занята другим сеансом — закрыть все окна 1С и попробовать снова
